# MedFlow POSEIDON (滴護寶) Gateway 燒錄治具與 SPI Flash 預燒錄設計說明書

本文件為 **MedFlow 滴護寶 (POSEIDON) Gateway 閘道器主板**（基於 ITE IT9866 SoC）之量產韌體燒錄設計規格。Gateway 系統使用一片 **GD25Q128ESIGR** (SOP8, 16MB) SPI Flash 作為系統啟動與韌體儲存媒介。針對工廠端量產需求，本設計提供兩種工業級燒錄方案：**方案 A：離線預燒錄（Gang Programming）** 與 **方案 B：在線測試針床燒錄（In-Circuit Programming, ICP/ISP）**。

---

## 1. 方案 A：離線預燒錄（量產主流，前段製程）

此方案是在 SMT 打件（表面貼裝）之前，使用工業級多路燒錄機將韌體包預先寫入空片 SOP8 晶片中。

### 1.1 離線燒錄設計細節
* **燒錄機選擇**：工廠通常使用 **1拖8 (8-Socket) 或 1拖4 (4-Socket) 的群組燒錄器**。
* **翻蓋式燒錄座 (SOP8 Clamshell Sockets)**：
  * 燒錄器上插接標準的 **SOP8 轉 DIP/鎖緊座轉接板**。
  * 對於 GD25Q128ESIGR，必須選用 **SOP8 200mil/208mil 寬體** 的翻蓋式燒錄座（注意：勿誤選為 150mil 窄體座）。
* **操作流程**：
  1. 作業員用真空吸筆或防靜電鑷子將空白 SOP8 SPI Flash 放進 8 個翻蓋座中並扣緊。
  2. 按下燒錄機上的實體 "Start" 鍵或在電腦端點擊並行寫入。
  3. 每個通道獨立燒錄與驗證（Verify），槽旁的 LED 指示燈轉為綠燈代表成功後即可取出，送往 SMT 生產線打件。

---

## 2. 方案 B：在線測試針床燒錄（ICP / ISP，後段製程）

此方案是在 Gateway 主板 SMT 打件完成後，使用**探針治具**直接在板子 上對 SPI Flash 進行韌體寫入或更新，適合生產後段的軟體升級或校準。

### 2.1 閘道器 PCBA 測試點 (TP) 設計
為配合在線燒錄，Gateway 的電路板（PCB）設計上必須預留 **6 個金屬測試環/點 (Test Pads)**，直接連往 SPI Flash 晶片的引腳：

```
[ Gateway 主板 PCB ]
   GD25Q128 (U1)          測試點 (TP)             治具端彈簧探針 (Pogo Pins)
   ┌──────────┐            ┌────┐                 ┌─────────────┐
   │ 1. /CS   ├───────────>│ TP1│ <───────────────┤ Pin 1 (CS#) │
   │ 2. DO(SO)├───────────>│ TP2│ <───────────────┤ Pin 2 (MISO)│
   │ 3. /WP   │ (接 VCC)   └────┘                 └─────────────┘
   │ 4. GND   ├───────────> 系統地 (GND)
   │ 5. DI(SI)├───────────>│ TP3│ <───────────────┤ Pin 3 (MOSI)│
   │ 6. CLK   ├───────────>│ TP4│ <───────────────┤ Pin 4 (SCK) │
   │ 7. /HOLD │ (接 VCC)   └────┘                 └─────────────┘
   │ 8. VCC   ├───────────>│ TP5│ <───────────────┤ Pin 5 (VCC) │
   └──────────┘            ┌────┐                 ┌─────────────┐
   IT9866 /RST ───────────>│ TP6│ <───────────────┤ Pin 6 (/RST)│
                           └────┘                 └─────────────┘
```

### 2.2 在線燒錄治具設計關鍵（防訊號衝突與隔離）
當外部燒錄器通過探針連接主板上的 SPI 匯流排時，為了防止與 IT9866 SoC 產生驅動衝突，治具必須進行**硬體隔離設計**：

1. **主晶片復位隔離 (SoC Reset Lock)**：
   * 治具增加一根探針壓接至 IT9866 的復位引腳（`/RST`，即上表中的 **TP6**），並將其**強制拉低接地 (Pull Low to GND)**。
   * 此操作會將 IT9866 SoC 鎖定在復位狀態，此時 SoC 的所有 GPIO（包含 SPI 控制引腳）都會轉為**高阻抗狀態 (Hi-Z)**，使外部燒錄器能完全接管 SPI 匯流排，避免訊號衝突導致燒錄失敗。
2. **電源二極體隔離**：
   * 治具上的供電探針（Pin 5）直接供電給 `+3.3V_SPI`。
   * 主板電路設計中，在系統 3.3V LDO 與 SPI Flash 供電腳之間串聯一顆蕭特基二極體或預留 0 歐姆電阻，防止治具的供電「倒灌」回主板的其他電路（如 SoC 核心），造成治具供電過載或主板部分通電。

### 2.3 治具結構與硬體配置
* **機械結構**：單槽電木治具，下底座有定位凹槽放置 Gateway 整塊 PCBA，上壓蓋透明壓克力板配置 **6 根彈簧探針** 對準板上的 TP1~TP6。
* **板載燒錄器 (In-System Programmer)**：
  * 治具內部直接安裝一台高速 USB 在線燒錄器（如 **DediProg SF600** 或 **SF600Plus**，或平價的 **CH341A USB-SPI 編程器**）。
  * 燒錄器經由同軸扁平線連接至探針板，並出線 USB 接至電腦。
  * 電腦端運行 DediProg 燒錄軟體（或命令行工具），可在主板不插電的狀態下，直接透過 USB 供電並在 10 秒內完成 16MB Flash 的寫入。

## 3. 方案對比與工廠建議

| 比較項目 | 方案 A：離線預燒錄 (SOP8 燒錄座) | 方案 B：在線針床燒錄 (ICP 治具) |
| :--- | :--- | :--- |
| **測試階段** | SMT 打件前 (裸晶片燒錄) | SMT 打件後 (PCBA 測試階段) |
| **生產速度** | **極高** (8路同時燒，打件即可開機) | **中等** (單片依序下壓燒錄) |
| **硬體要求** | 需 SOP8 200mil 翻蓋燒錄轉接座 | PCB 需設計 6 個測試點 (TP) 及隔離電路 |
| **後續維護** | 若需更新韌體，必須拆卸晶片重新燒錄 | **極佳** (後續更新只需重新下壓治具即可) |
| **設備成本** | 需配備專業多路燒錄機 (Elnec/DediProg) | 需客製探針治具 + 板載 SF600 燒錄器 |
| **工廠建議** | **首選量產方案**。第一版量產韌體（包含 Bootloader）建議全部採用預燒錄，可保證產線流速。 | **研發與首批試產 (Pilot Run) 推薦**。或於後段測試站作為「韌體升級與校正數據寫入」使用。 |
