# MedFlow 尿袋監測器設計規格與操作 SOP (Urine Bag Monitor Specs)

本文件記錄 **尿袋 IO (Smart Urine Bag Monitor)** 的硬體元件設計、GPIO 腳位配置、藍牙廣播行為、閘道器上報規格，以及生產操作 SOP 的檔案路徑。

---

## 1. 系統三大核心元件 (Three Core Components)

尿袋監測系統由以下三個主要元件協同運作：

### 1.1 貼片感測端 (Sensor Patch)
* **硬體組成**：電容感測貼片、**Touch IC**、**CR2032 鈕扣電池**。
* **運作機制**：利用電容感應液位變化，當尿液高度到達感測區時，輸出數位信號給藍牙端點：
  * **`1`**：代表 **尿袋空** (Empty)
  * **`0`**：代表 **尿袋滿** (Full)

### 1.2 藍牙端點 (BT Node - ATB1113)
* **晶片方案**：採用炬芯科技 (Actions) **ATB1113 IC**。
* **設計資料路徑**：[C:/公司/Design/睿建滴護寶/ATB1113](file:///C:/公司/Design/睿建滴護寶/ATB1113)
* **電源供應**：由貼片上的 CR2032 電池直接供電。
* **腳位配置 (GPIO Configuration)**：
  * **GPIO8**：**輸入引腳**，接收貼片輸出的 `1` 或 `0` 訊號。
  * **GPIO21 & GPIO20**：**輸出引腳**，控制不同的 LED 指示燈號，用以顯示目前為「空」或「滿」狀態。
* **廣播頻率控制**：內建使用 **RTC** (Real-Time Clock) 精確控制藍牙廣告廣播 (Advertising) 頻率，確保低功耗運作，同時使 Gateway 能接收到即時的尿袋狀態。
* **床位關聯**：廣播時的 **MAC 位址 (MAC Address)** 代表對應的 **床位**，後台雲端伺服器會依此 MAC 位址進行病床綁定與中控顯示。

### 1.3 閘道器端 (Gateway)
* **運作機制**：掃描並接收區域內尿袋 BT 端點的廣播封包，將其打包後上傳至雲端伺服器。
* **上報資料內容**：
  * 閘道器自身名稱 (Gateway Name)。
  * 接收到的藍牙廣播資料（包括：產品型號、尿袋狀態）。
  * 遙測定位數據：端點 MAC、訊號強度 (**RSSI**)、接收日期時間 (Timestamp) 或 Wi-Fi MAC 位址。
* **覆蓋率辨識**：透過收集這些含有 RSSI 與 Gateway 名稱的封包，雲端平台可以分析並評估 **Gateway 在醫院/照護中心現場的訊號涵蓋率 (Coverage)** 是否足夠。
